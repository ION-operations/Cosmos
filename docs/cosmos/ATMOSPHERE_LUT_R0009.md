# R-0009 physical atmosphere LUT interface

R-0009 starts the physical-atmosphere path for Cosmos without replacing the full renderer in one step.

The goal is a stable **ground-to-space atmosphere lookup contract** that every major pass can sample:

```text
transmittance LUT
multi-scattering LUT
sky-view LUT
aerial-perspective LUT
optical-depth debug channel
```

This patch deliberately separates the interface from the eventual high-accuracy solver. The current textures are generated by a deterministic CPU fallback so that WebGL routes, UI controls, tests, and diagnostics are stable before a heavier Bruneton/Hillaire-style precomputation pipeline is integrated.

## New runtime module

```text
src/cosmos/atmosphere/atmosphereLut.ts
```

Exports:

```text
COSMOS_ATMOSPHERE_LUT_SPEC
DEFAULT_COSMOS_ATMOSPHERE_LUT_CONTROLS
sampleAtmosphereOpticalModel(...)
createCosmosAtmosphereLutTextures(...)
createCosmosAtmosphereLutState(...)
disposeCosmosAtmosphereLutTextures(...)
```

## LUT textures

| Texture | Resolution | Encoding |
|---|---:|---|
| transmittance | 256×64 | RGB=solar transmittance, A=normalized optical depth |
| multiScattering | 64×32 | RGB=multiple-scattering tint, A=energy |
| skyView | 192×108 | RGB=sky-view radiance proxy, A=horizon/terminator |
| aerialPerspective | 96×64 | RGB=aerial-perspective tint, A=fog density |

## Runtime controls

`/cosmos-review` now exposes:

```text
LUT strength
Rayleigh scale
Mie/aerosol scale
Ozone scale
Multiple scattering
Aerial perspective
Sky-view LUT
Optical debug
```

The route also exposes:

```js
window.__COSMOS_ATMOSPHERE_LUT_STATE__
```

## New debug mode

R-0009 adds debug mode 6:

```text
optical-depth-lut
```

Colour contract:

```text
R = optical depth
G = aerial perspective
B = multi-scattering / orbital rim contribution
```

## Current limitation

The R-0009 LUTs are deterministic **interface/fallback** textures, not yet the final physical solver.

The next step is to run the browser shader path locally and capture WebGL shader compile logs, beauty screenshots, debug overlays, and twilight/terminator validation views. That matters because Vite can validate TypeScript while still missing runtime GLSL compile failures.
