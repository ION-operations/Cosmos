## What Is Prepared

- Static hosted landing page source at `pages/`.
- Public Vite landing route at `/` for local app runs.
- Interactive renderer at `/lab` for local app runs.
- Existing review tooling at `/cosmos-review` for local app runs.
- Local run console at `/cosmos-local-run` for local app runs.
- All seven R0004 bookmark IDs are linked from the public page: `orbit`, `cloud-terminator`, `high-altitude`, `storm-zone`, `sun-glitter`, `sea-level`, `underwater`.
- Vite base path still switches to `/Cosmos/` when `GITHUB_PAGES=true` for future full-app hosting.
- GitHub Actions Pages workflow at `.github/workflows/pages.yml`.
- Hosted Pages artifact generated from `pages/`, `public/cosmos/pages`, `public/cosmos/gibs`, and `public/cosmos/bathymetry`.
- Fallback generated by copying `dist/index.html` to `dist/404.html`.

## Required GitHub Setup

1. Create or connect the repository as `ION-operations/Cosmos`.
2. Push this project to `main`.
3. In repository settings, set Pages source to **GitHub Actions**.
4. Run **Deploy Cosmos GitHub Pages**.

## Design Notes

The hosted page uses the R0004 review captures as real visual assets, not abstract decoration. The design follows the current ION README pattern:

- plain-language hero;
- evidence and status tables;
- architecture/purpose summary;
- validation instructions;
- limits and next-candidate work stated directly.
