```text
ION-operations/Cosmos
```

Target page:

```text
https://ion-operations.github.io/Cosmos/
```

## What Is Prepared

- Public landing route at `/`.
- Interactive renderer at `/lab`.
- Existing review tooling at `/cosmos-review`.
- Local run console at `/cosmos-local-run`.
- All seven R0004 bookmark IDs are linked from the public page: `orbit`, `cloud-terminator`, `high-altitude`, `storm-zone`, `sun-glitter`, `sea-level`, `underwater`.
- Vite base path switches to `/Cosmos/` when `GITHUB_PAGES=true`.
- GitHub Actions Pages workflow at `.github/workflows/pages.yml`.
- SPA fallback generated by copying `dist/index.html` to `dist/404.html`.

## Required GitHub Setup

1. Create or connect the repository as `ION-operations/Cosmos`.
2. Push this project to `main`.
3. In repository settings, set Pages source to **GitHub Actions**.
4. Run **Deploy Cosmos GitHub Pages**.

## Design Notes

The page uses the R0004 review captures as real visual assets, not abstract decoration. The design follows the current ION README pattern:

- plain-language hero;
- evidence and status tables;
- architecture/purpose summary;
- validation instructions;
- limits and next-candidate work stated directly.

## Current Access Note

The local GitHub token could not query `ION-operations` during preparation. GitHub returned a 404/scope hint for org access. Remote creation/deployment may require refreshing the token with the correct organization permissions or creating the repository manually from the GitHub UI.
